Iguana

Iguana is an open source issue management system with a kanban board, sprint planning
and time logging features, and anything else you need to make working on a project in groups pleasant.

You can get in on github.

API documentation

endpoints:
  • api/users/(GET) (1)
  • api/users/{username}/ (GET) (2)
  • api/timelogs/ (GET) (3)
  • api/issues/ (GET) (4)
  • api/projects/ (GET, POST) (5)
  • api/projects/{project_short_name}/ (GET, PUT, PATCH, DELETE) (6)
  • api/projects/{project_short_name}/issues/ (GET, POST) (7)
  • api/projects/{project_short_name}/issues/{issue_number}/ (GET, PUT, PATCH, DELETE) (8)
  • api/projects/{project_short_name}/issues/{issue_number}/timelogs/ (GET, POST) (9)
  • api/projects/{project_short_name}/issues/{issue_number}/timelogs/{timelog_number}/ (GET, PUT, PATCH, DELETE) (10)
  • api/projects/{project_short_name}/issues/{issue_number}/comments/(GET, POST) (11)
  • api/projects/{project_short_name}/issues/{issue_number}/comments/{comment_number}/ (GET, PUT, PATCH, DELETE) (12)
  • api/projects/{project_short_name}/timelogs/ (GET) (13)
restrictions:
  • request.user has to be authenticated (1-12)
  • request.user has to be in project (6-13)
  • request.user has to be project manager (6(PUT/PATCH/DELETE))
  • request.user has to be owner(10/12(PUT/PATCH/DELETE))
authentication:
  • HTTP Basic Authentication
filters:
  • Issuefilter(endpoints: 4,7)
    • status(exact)
      ?status=Done
    • assignee(contains)
      ?assignee=user1&assigne=user2
    • participant(contains)
      ?participant=user1
    • dependsOn(exact)
      ?dependsOn=9 (makes only sense in project context, issue numbers are only unique in project )
    • sprint(exact)
      ?sprint=5 (makes only sense in project context, sprint numbers are only unique in project )
    • project(exact, short_name)
      ?project=TPJ
    • logged_total(gt, lt, exact)
      ?logged_total__gt=2h30m&logged_total__lt=3h30m or ?logget_total=2h
    • priority(gt, lt, exact)
      ?priority__gt=1 or ?priority=4
    • type(exact)
      ?type=Bug or ?type=Task
    • due_date(exact, lt, gt)
      ?due_date__lt=2017-05-05T16:47:00Z
    • created(exact, lt, gt)
      ?created__gt=2017-04-03T00:00:00Z
    • storypoints(exact, lt, gt)
      ?storypoints__gt=4&storypoints__lt=11
    • archived(exact)
      ?archived=True
  • TimelogFilter(endpoints: 3, 9, 13)
    • time(exact, lt, gt)
      ?time__gt=2h15m&time__lt=3h
    • user(exact)
      ?user=user1
    • created_at(exact, lt, gt)
      ?created_at__gt=2017-04-03T00:00:00Z
